# ClaudeCodeBrowser Firefox Integration Design

**Date**: 2026-02-28
**Status**: Approved
**Branch**: firefoxclaude

## Goal

Integrate [ClaudeCodeBrowser](https://github.com/nanogenomic/ClaudeCodeBrowser)
into the dotfiles so Claude Code can automate Firefox — screenshots, clicking,
typing, navigation, and page interaction. Install into the primary Firefox
profile with CORS hardening applied.

## Architecture

```
Claude Code CLI
    │
    ├── MCP stdio ──> ~/.claudecodebrowser/mcp-server/server.py
    │                      │
    │                      ├── HTTP (localhost:8765) ──> command queue
    │                      └── WebSocket (localhost:8766) ──> real-time
    │
    └── (MCP response) <── Native Messaging Host (stdio)
                               │
                               └── Firefox Extension (content.js + background.js)
                                      │
                                      └── Web Page (DOM, screenshots, interaction)
```

**Communication flow**:
1. Claude Code calls MCP tool (e.g., `browser_screenshot`)
2. MCP server queues the command
3. Firefox extension polls for commands (HTTP) or receives via WebSocket
4. Extension executes in browser context
5. Result flows back through native messaging host → MCP server → Claude Code

## Components

### 1. Repository Clone (`~/.claudecodebrowser/`)

Clone the repo to a well-known path outside `~/dotfiles` (similar to
`~/.openclaw/` pattern — runtime state, not stow-managed).

```
~/.claudecodebrowser/
├── native-host/
│   ├── claudecodebrowser_host.py    # Native messaging bridge
│   └── claudecodebrowser.json       # Manifest template (Linux)
├── mcp-server/
│   ├── server.py                    # MCP HTTP/WS server (CORS-patched)
│   ├── stdio_wrapper.py             # stdio wrapper for Claude Code
│   └── mcp_config.json              # MCP config
├── extension/
│   ├── manifest.json                # Firefox extension manifest
│   ├── background.js                # Extension background script
│   ├── content.js                   # Content script (injected into pages)
│   └── ...
├── agent/                           # High-level automation agent
├── screenshots/                     # Captured screenshots
└── logs/                            # Server logs
```

### 2. CORS Hardening

Patch `server.py` to remove the wildcard CORS header:

```python
# Before (vulnerable — any webpage can reach the MCP server):
self.send_header('Access-Control-Allow-Origin', '*')

# After (hardened — only localhost can reach it):
# Remove the header entirely, or restrict to null origin.
# The native messaging host communicates via stdio, not HTTP,
# so no CORS header is needed for extension communication.
```

The patch is applied via `sed` in setup.sh so it's reproducible and survives
`ccb update` (re-applied after each pull).

### 3. Native Messaging Host (macOS)

Firefox on macOS looks for native messaging manifests at:
```
~/Library/Application Support/Mozilla/NativeMessagingHosts/claudecodebrowser.json
```

The manifest content (generated by setup.sh with correct path):
```json
{
  "name": "claudecodebrowser",
  "description": "ClaudeCodeBrowser Native Messaging Host",
  "path": "/Users/<username>/.claudecodebrowser/native-host/claudecodebrowser_host.py",
  "type": "stdio",
  "allowed_extensions": ["claudecodebrowser@ligandal.com"]
}
```

### 4. MCP Server Configuration (Parity Rule)

**Claude Code CLI** (via `claude mcp add` in setup.sh):
```bash
claude mcp add claudecodebrowser \
  --transport stdio \
  -- python3 "$HOME/.claudecodebrowser/mcp-server/stdio_wrapper.py"
```

**Claude Desktop** (in `claude_desktop_config.json`, stow-managed):
```json
{
  "claudecodebrowser": {
    "command": "python3",
    "args": ["/Users/<username>/.claudecodebrowser/mcp-server/stdio_wrapper.py"]
  }
}
```

### 5. Firefox Extension Installation

Installed from [AMO](https://addons.mozilla.org/en-US/firefox/addon/claudecodebrowser/)
(Firefox Add-ons store). This is the only manual step — Firefox doesn't allow
programmatic extension installation.

Setup.sh prints the URL with instructions after completing automated setup.

### 6. Fish Function (`ccb`)

Location: `.config/fish/functions/ccb.fish`

```
ccb start    — Launch MCP server in background, track PID
ccb stop     — Kill MCP server by PID
ccb status   — Show server running state, port availability, extension ID
ccb logs     — Tail ~/.claudecodebrowser/logs/
ccb update   — git pull + re-apply CORS patch + restart if running
```

### 7. Python Dependencies

```bash
pip3 install websockets    # Required for WebSocket server (port 8766)
```

Alternatively, the system works in HTTP-only mode without websockets (the
extension falls back to polling). We install it for better performance.

## Setup Script Changes (`scripts/setup.sh`)

### Phase 3 — Development Tools

```bash
pip3 install websockets
```

### Phase 4 — Cloud & Infrastructure (new section)

```bash
# ClaudeCodeBrowser - Firefox browser automation for Claude Code
CCB_DIR="$HOME/.claudecodebrowser"
if [ ! -d "$CCB_DIR" ]; then
  git clone https://github.com/nanogenomic/ClaudeCodeBrowser.git "$CCB_DIR"
else
  cd "$CCB_DIR" && git pull && cd -
fi

# CORS hardening - remove wildcard origin
sed -i '' "s/Access-Control-Allow-Origin', '\*'/Access-Control-Allow-Origin', 'null'/" \
  "$CCB_DIR/mcp-server/server.py"

# Make scripts executable
chmod +x "$CCB_DIR"/native-host/*.py "$CCB_DIR"/mcp-server/*.py

# Register native messaging host (macOS)
NMH_DIR="$HOME/Library/Application Support/Mozilla/NativeMessagingHosts"
mkdir -p "$NMH_DIR"
cat > "$NMH_DIR/claudecodebrowser.json" << NMHEOF
{
  "name": "claudecodebrowser",
  "description": "ClaudeCodeBrowser Native Messaging Host",
  "path": "$CCB_DIR/native-host/claudecodebrowser_host.py",
  "type": "stdio",
  "allowed_extensions": ["claudecodebrowser@ligandal.com"]
}
NMHEOF

# MCP server registration (Claude Code CLI)
claude mcp add claudecodebrowser \
  --transport stdio \
  -- python3 "$CCB_DIR/mcp-server/stdio_wrapper.py"

# Reminder for manual Firefox extension install
echo "📦 ClaudeCodeBrowser: Install Firefox extension from:"
echo "   https://addons.mozilla.org/en-US/firefox/addon/claudecodebrowser/"
```

### Claude Desktop Config Update

Add to `Library/Application Support/Claude/claude_desktop_config.json`:

```json
"claudecodebrowser": {
  "command": "python3",
  "args": ["~/.claudecodebrowser/mcp-server/stdio_wrapper.py"]
}
```

## Hardening Applied

| Risk | Mitigation |
|------|-----------|
| CORS wildcard | Patched to `null` origin — no webpage can reach MCP server |
| No authentication | Localhost-only bind (default `127.0.0.1`), CORS restricted |
| Supply chain | Source cloned from GitHub, can pin to specific commit |
| Native host tampering | Installed by setup.sh to known path, can hash-verify |
| Extension auto-update | AMO-managed updates (Mozilla-reviewed) |

## What We're NOT Doing

- **Separate Firefox profile**: User chose primary profile integration
- **Commit pinning**: User opted against SHA pinning (can add later)
- **Firewall rules**: `pf` rules are optional hardening, not default
- **Brewfile entry**: No Homebrew package exists for this; git clone is sufficient

## Testing

After installation:
1. `ccb start` — server should start on ports 8765/8766
2. `ccb status` — should show server running
3. Open Firefox → extension icon should be visible
4. In Claude Code: `mcp__claudecodebrowser__browser_screenshot` should work
5. `ccb stop` — server should stop cleanly
